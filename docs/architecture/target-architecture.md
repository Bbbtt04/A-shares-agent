# 股票情报 Agent 总架构

本文档定义股票情报 Agent 的长期目标架构。系统定位为面向 A 股的情报分析与策略辅助系统，优先提供可追溯、可解释、可复盘的盘前与盘中决策支持；默认不直接接入真实交易执行。

## 1. 架构目标

核心目标：

- 支持盘前策略生成、盘中异动解释、盘后复盘闭环。
- 将业务 Agent 与基础能力解耦，避免每个 Agent 重复实现模型调用、工具权限、数据查询和监控逻辑。
- 所有重要结论必须有证据链，能够区分事实、推断、观点和风险。
- 盘中链路优先低延迟与稳定性，盘前链路优先完整性与证据充分性。
- 支持后续从情报辅助系统逐步扩展到模拟交易、人工审批交易或更高阶自动化，但默认保持安全隔离。

## 2. 总体分层

```text
数据源
  -> 数据接入与治理层
  -> 知识与状态层
  -> 基建能力层
  -> Agent 编排层
  -> 专业 Agent 层
  -> 产品与交互层
```

### 2.1 数据源

数据源包括：

- 行情数据：实时行情、历史 K 线、盘口、成交、板块指数、涨跌停、资金流。
- 新闻资讯：财经新闻、行业新闻、政策新闻、突发事件、社媒舆情。
- 上市公司公告：定期报告、临时公告、减持、增持、回购、并购重组、监管函、风险提示。
- 财务与基本面：财报、估值、行业对比、主营构成、股东结构。
- 研报与机构观点：行业研报、个股研报、策略研报、盈利预测。
- 用户数据：自选股、关注行业、策略偏好、提醒反馈。

### 2.2 数据接入与治理层

数据治理是系统质量上限的关键模块，职责包括：

- 多源接入、数据清洗、去重与归一化。
- 股票代码、公司名称、简称、曾用名、概念标签的统一映射。
- 新闻发布时间、事件发生时间、行情时间的时间对齐。
- 数据延迟、缺失、异常值和重复事件检测。
- 数据血缘记录，保留来源、抓取时间、处理版本和可信度。
- 对重要数据源建立健康检查与降级策略。

### 2.3 知识与状态层

知识与状态应分开设计。

长期知识包括：

- 公司知识库：主营业务、历史公告、财务质量、风险事件、股性特征。
- 行业知识库：产业链、上下游、政策周期、竞争格局、核心指标。
- 题材知识库：概念定义、核心标的、历史炒作路径、扩散规律。
- 策略知识库：历史策略、复盘结论、信号有效性、失败案例。
- 制度知识库：交易规则、监管规则、财报规则、公告规则。

短期状态包括：

- 今日市场主线。
- 今日关注股票池。
- 盘前策略假设。
- 个股当前状态。
- 盘中已触发提醒。
- 用户反馈与人工标记。
- Agent 判断历史与复盘结果。

RAG 知识条目必须包含来源、发布时间、生效时间、关联实体、可信度、过期状态和证据链接。股票情报中时间属性非常重要，过期知识不能被当成当前事实。

### 2.4 基建能力层

基建层为所有 Agent 提供统一能力。

#### 沙盒与权限

沙盒负责限制 Agent 能做什么、能访问什么、能写入什么。

建议权限分层：

- 只读权限：查询行情、新闻、公告、知识库。
- 分析权限：调用模型、运行计算、生成报告。
- 写入权限：更新画像、写策略状态、写复盘记录。
- 通知权限：发送飞书、企业微信、邮件或 App 推送。
- 高危权限：接入交易接口、修改策略配置、批量发送高优先级提醒。

每个 Agent 应有独立 permission profile，并记录所有工具调用与写入行为。

#### LLM 网关

LLM 网关负责统一模型访问，不允许业务 Agent 直接裸调模型。

职责包括：

- 多模型路由。
- Prompt 模板管理。
- Token 预算与成本统计。
- 请求缓存、重试、超时和降级。
- 结构化输出校验。
- 模型版本管理。
- 调用日志与审计。
- 敏感输出过滤。

不同任务可使用不同模型：新闻摘要使用低成本模型，公告解析使用结构化能力强的模型，复杂策略推理使用强推理模型，盘中提醒使用低延迟模型。

#### 工具管理

工具管理层采用 Tool Registry 设计。

每个工具需要定义：

- 名称与描述。
- 输入 schema 与输出 schema。
- 所需权限。
- 超时、重试、频率限制。
- 成本估算。
- 是否可缓存。
- 是否需要审计。
- 失败时的 fallback。

工具类型包括行情工具、资讯工具、公告工具、财务工具、RAG 工具、计算工具、通知工具和报告工具。工具输出应尽量结构化，避免只返回自然语言。

#### 监控与审计

监控分为系统监控、Agent 监控和业务监控。

系统监控：

- API 延迟。
- 任务队列堆积。
- 数据源延迟。
- 工具调用成功率。
- LLM 调用失败率。
- Token 成本。

Agent 监控：

- 调用次数、错误率、超时率。
- 输出格式错误率。
- 空输出比例。
- 证据缺失比例。
- 提醒命中率。
- 盘前观点验证比例。

业务监控：

- 每日情报数量。
- 用户点击、收藏、反馈。
- 高优先级提醒数量。
- 有效信号来源分布。
- 数据源贡献度。

### 2.5 Agent 编排层

编排层负责把多个 Agent、工具和状态串成稳定工作流。

职责包括：

- 定时任务。
- 事件触发。
- Agent 调用顺序。
- 上下文传递。
- 状态管理。
- 失败重试。
- 并发控制。
- 人工确认节点。

盘前典型流程：

```text
拉取隔夜数据
  -> 新闻聚类
  -> 公告解析
  -> 板块强度分析
  -> 个股池更新
  -> 盘前策略生成
  -> 风险检查
  -> 盘前报告生成
```

盘中典型流程：

```text
行情异动触发
  -> 判断异动真实性
  -> 查询新闻、公告、板块联动
  -> 匹配盘前策略假设
  -> 输出解释与提醒
  -> 更新标的状态
```

## 3. 专业 Agent 层

### 3.1 盘前策略 Agent

目标是生成今日交易前情报包与策略假设。

输入：

- 隔夜外盘、商品、汇率、利率。
- 国内政策、宏观、行业新闻。
- 个股公告、财报、资金流、龙虎榜。
- 昨日市场结构、涨停板、题材强度。
- 用户关注股票池。

输出：

- 今日市场环境判断。
- 主线题材排序。
- 风险提示。
- 关注股票池与回避股票池。
- 每只股票的催化、证据、风险、观察条件。
- 可被盘中 Agent 使用的策略假设。

### 3.2 盘中策略 Agent

目标是实时解释市场异动，并判断盘前假设是否被验证或证伪。

输入：

- 实时行情、成交、盘口、涨速、换手。
- 板块联动。
- 突发新闻与公告。
- 盘前策略上下文。
- 当前关注池状态。

输出：

- 异动解释。
- 机会提醒。
- 风险提醒。
- 标的状态流转。
- 盘前策略更新。

盘中输出应使用分级提醒，例如 P0 重大风险、P1 强异动、P2 一般跟踪、P3 背景信息。

### 3.3 新闻情报 Agent

负责多源新闻抓取、去重、聚类、可信度评级、事件时间线和影响标的识别。

### 3.4 公告解读 Agent

负责解析上市公司公告，提取关键数值与事件类型，判断是否超预期，并和历史公告、财务数据对比。

### 3.5 行情异动 Agent

负责识别涨速、放量、跌幅、炸板、回封、板块联动和资金流变化，是盘中链路的核心触发器之一。

### 3.6 板块与题材 Agent

负责识别题材强度、龙头、补涨、后排、扩散路径和退潮风险。

### 3.7 个股画像 Agent

负责维护公司基本面、概念标签、历史股性、风险标签、题材参与度和机构关注度。

### 3.8 策略复盘 Agent

负责盘后复盘，评估盘前判断与盘中提醒是否有效，归因错误来源，并沉淀到策略知识库。

## 4. 风控与合规原则

系统必须显式区分：

- 事实：已发生且可验证的信息。
- 推断：基于多个信号形成的分析。
- 观点：Agent 的策略判断。
- 风险：可能导致判断失效的因素。

默认原则：

- 不承诺收益。
- 不把未经验证的传闻当事实。
- 重要结论必须带证据。
- 高风险标的必须标注风险原因。
- 真实交易接口与情报 Agent 隔离。
- 高危动作需要人工确认。

## 5. 评测体系

评测系统应覆盖：

- 结构化输出稳定性。
- 新闻聚类与事件归因准确率。
- 公告字段抽取准确率。
- 个股关联准确率。
- 异动解释合理性。
- 策略观点过度自信比例。
- 提醒后市场表现。
- 人工反馈分数。

建议建立黄金样本集：

- 历史重大行情日。
- 典型公告案例。
- 典型题材炒作案例。
- 典型误导新闻案例。
- 典型高开低走案例。
- 典型炸板与回封案例。

## 6. 产品形态

建议产品层包含：

- 盘前日报。
- 盘中实时提醒。
- 个股问答。
- 板块追踪页。
- 关注池。
- 策略假设看板。
- 盘后复盘日报。
- 数据源与证据溯源页面。

## 7. 长期演进路线

建议分阶段推进：

1. 情报辅助：盘前报告、盘中异动解释、盘后复盘。
2. 模拟交易：在严格风控下产生 paper trading intent 并复盘。
3. 人工审批交易：高置信信号进入审批队列，由人确认。
4. 更高阶自动化：只有在评测、风控、审计和合规能力成熟后再考虑。

系统最重要的长期原则是：先把数据、证据、评测和复盘做好，再提高 Agent 自主性。
